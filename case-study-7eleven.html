◆ Context

six roles, one operational backbone

7-Eleven's operations span thousands of store locations, each generating data that multiple roles depend on — but historically, each role used disconnected tools with no shared visual language or interaction model.

Every operational signal — sales, inventory, appliance health, tickets — flows up through the same underlying system. This made the design system itself the highest-leverage surface, where consistency compounds across six distinct experiences.

The business wanted faster issue resolution and clearer accountability, while frontline managers reported alert fatigue and tool-switching fatigue.

◆ Key Decisions

four calls that shaped the system

📌
01
One design system, six personas
Rather than designing each experience independently, we built a shared component library first. The same card tiles, KPI blocks, and navigation sidebar became the common foundation — then role-specific information architecture was layered on top. This cut design handoff time and made future changes platform-wide, not per-product.
📌
02
Severity should drive color, not decoration
We created a strict semantic color system: red for Critical, orange for High, green for Good/Excellent — applied consistently across every dashboard, table, and status pill. Color became signal, not style, so any user could instantly parse urgency without reading the label.
📌
03
Corporate needs synthesis, not more data
The Corporate Manager view ($56.2M total sales, 8 stores at risk, 42 IoT alerts) could easily become a data graveyard. We built a Corporate Action Center that surfaces a prioritized action queue — the highest-impact issues requiring a decision right now — rather than dumping all available metrics at once.
📌
04
Security should feel like part of the product
The Password Management portal and IT Admin Policy Engine share the exact same sidebar nav, card patterns, and status system as the operational dashboards. When an IT admin opens a reset request, it looks like the same product family — not a legacy admin panel dropped in from a different era.

◆ Multi-Persona System

one system, six different jobs

The Corporate Manager dashboard aggregates signals from every layer of the operation — store-level IoT alerts, supply chain gaps, sales vs. target variance — into a single synthesized view.

Rather than building a custom data model for each persona, we asked: what does this role need to decide right now? That question shaped the information hierarchy, and the shared design system made each answer feel native to the same product.

The result: VPs can spot a revenue risk before it becomes a reporting issue. Area managers can act on a single store without losing sight of their portfolio.

◆ Security UX

security that doesn't feel like security

The IT Admin panel had to enforce strict compliance — PCI-DSS, SOX, MFA mandates, executive-level password policies — without feeling like a bolted-on enterprise tool.

We brought the same card-based KPI system (22 active rules, 7 locked accounts, 5 high-risk credentials) into the security layer, and expressed complex policy logic as human-readable IF/THEN conditions:

IF daysSinceLastChange < 7 THEN BLOCK
IF role == "executive" THEN REQUIRE_MFA
IF failedAttempts > 3 THEN LOCK + NOTIFY
// PCI-DSS Compliance · Critical priority

Admins see the logic they authored reflected back as clear rules — not encoded in a policy engine only they can read.

◆ Reflections
📎

What I learned building this

  • The hardest part wasn't any single screen — it was making six genuinely different jobs feel like one product.
  • Severity and status need one shared visual vocabulary, or trust breaks down fast. When "red" means different things in different portals, users stop trusting the system.
  • Treating security and admin tools with the same design rigor as the "real" product builds confidence across the entire platform — not just for IT admins.
  • A design system only proves itself once it's stress-tested across genuinely different user needs. Ours held up because we challenged it early with the most unlike personas we had.
